      Once you have anonymous functions in the abstract
	    representation —
	    the EFunction expression node — we no
	      longer need ELet in the abstract
	      representation. We can have let in the surface
	    syntax as usual, but use a front-end
		transformation to transform any let into
	      uses of anonymous functions and function calls. (See Homework 4 for front-end transformations.)
      

	    
      You should convince yourself that
	      
        (let ((x (* 2 2))
        (y 20))
    (+ x y))

      
	      could be written
	      
        ((function (x y) (+ x y))
      (* 2 2) 20)

      
	Note the parenthesization — this is a function expression applied to two arguments.
	      More generally,

        (let ((n1 E1) (n2 E2) ...)
     F)

      
	      can always be written as

        ((function (n1 n2 ...) F)
      E1 E1 ...)

      
	    
      

	    
      Implement the above transformation in the parser
	      function parse. Note that the current version
	      of the parser raises an exception if you try to parse
	      a let.

    2. It turns out we don't need multi-argument
	      functions. If you have single-argument anonymous functions,
	      you can code up multiple argument functions using a
	      technique known
	      as currying
	      (named after Haskell Curry, not the Indian dish).
      

	    
      Here's the idea. Suppose you wanted to define a
	      function taking two arguments, such as

        (defun foo (x y) E)

      
You call such a function with (foo 10 20).
      

	    
      Another way to write that function is as follows:

        (defun foo-alternate (x)
    (function (y) E))

      
	      namely as a function that takes the first argument as
	      input, and returns a function that takes the second
  argument as input and returns the final result. To
	      invoke such a function, you would use:
	      ((foo-alternate 10) 20)
	      Make sure you understand what's going on here. If you
	      do, then you've understood first-class functions.
	    
      

	    
      Here is how you would write a three-arguments function:

        (defun bar-alternate (x)
    (function (y)
      (function (z) F)))

      
	      and you would call it as (((bar-alternate 10) 20)
		30).
	    
      

	    
      Write a version of the parser called parse_curry
	      that accepts the same surface syntax as your parser in Question 1 (multi-arguments functions and calls) but transforms multi-argument functions into single-argument function as above, and multi-argument function calls into nested function calls, again as above. Basically, you should accept the same surface syntax as Question 1, but you should produce an abstract representation that never uses functions (or closures) with more than one argument.
      

	    
      I've given you a companion shell shell_curry that implements a shell just like shell except using parse_curry. 
	      It uses initial_env_curry, which creates an initial
	      environment that already contains function wrappers for our
	      primitive operations that uses curried functions.
