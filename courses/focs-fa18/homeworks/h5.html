Here is a type for deterministic Turing machines, parameterized by the type for states.

type symbol = string

type 'a tm = { states : 'a list;
	       input_alphabet : symbol list;
	       tape_alphabet : symbol list;
	       left_marker : symbol;
	       blank : symbol;
	       delta : ('a * symbol) -> ('a * symbol * int);
	       start : 'a;
	       accept : 'a;
	       reject : 'a }

The definition follows that of Turing machines we've seen in class. Note that the transition function is given by an OCaml function taking as input a pair (p,a) of the current state and the symbol under the tape head and returning a triple (q,b,dir) indicating that the machine should transition into a state q, writing b in the cell under the tape head, and then moving the tape head in direction dir, left when it is 0, and right when it is 1.

In OCaml, like in most modern languages like Python and Javascript, functions are values like any other — they can be put into lists and records, they can be passed as arguments to other functions, they can be returned from other functions. For example, here is a function that takes a one-argument function as an argument as well as an integer, and applies the function twice in succession to the integer:

# let twice (f:int -> int) (x:int):int = f (f x);; 
val twice : (int -> int) -> int -> int = <fun>

# let double x = 2 * x;;
val double : int -> int = <fun>

# double 10;;
- : int = 20

# twice double 10;;
- : int = 40

# let square x = x * x;;
val square : int -> int = <fun>

# square 20;;
- : int = 400

# twice square 20;;
- : int = 160000

Another difference between our Turing machines implementation and that of finite automata is that symbols are no longer characters, but rather strings (though I'm using the name symbol to refer to them for clarity). That's because we may want to mark up elements of the alphabet, and it's easier to mark up symbols when they are represented as strings instead of characters. I've given you a function explode that takes a string and explodes it into a list of constituent symbols.

As a simple example, here is the code implementing a Turing machine accepting the regular language {ambn | m,n≥0}:

let asbs =
  let d inp = match inp with
    | ("start", "a") -> ("start", "a", 1)
    | ("start", "b") -> ("q1", "b", 1)
    | ("start", ">") -> ("start", ">", 1)
    | ("start", "_") -> ("acc", "_", 1)
    | ("q1", "b") -> ("q1", "b", 1)
    | ("q1", "_") -> ("acc", "_", 1)
    | ("acc", "a") -> ("acc", "a", 1)
    | ("acc", "b") -> ("acc", "b", 1)
    | ("acc", ">") -> ("acc", ">", 1)
    | ("acc", "_") -> ("acc", "_", 1)
    | (_,c) -> ("rej",c,1)
  in
    { states = ["start"; "q1"; "acc"; "rej"];
      input_alphabet = ["a";"b"];
      tape_alphabet = ["a";"b";"_";">"];
      blank = "_";
      left_marker = ">";
      start = "start";
      accept = "acc";
      reject = "rej";
      delta = d }

Note the function d defined locally and placed inside the record.

Recall from lecture that a configuration captures the current state of a Turing machine, the content of the tape, and the position of the tape head. Type 'a config is the type of a configuration, defined as follows:

type 'a config = { state : 'a;
                   tape : symbol list;
                   position : int }

Field state is the current state, field tape is the content of the tape, and position is the position of the tape head, with 0 as the left-most position on the tape. A configuration should always include enough symbols in tape to include the given position, possibly by padding the end of the tape list with blank symbols (as defined by the machine description).
