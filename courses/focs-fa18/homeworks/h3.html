  1. Language operations

    A language is a set of strings, where a set is a collection of elements in which repetition and order is irrelevant; the languages {"a","a","b","c"} and {"b","c","c","a"} are considered equal.

    We use lists as a representation for sets, and therefore languages. A language will have type string list. We define language operations that work with that list representation. You have a choice to make for that representation though: do you get rid of repeated strings in the representation, or do you keep them in and make sure that functions to determine, say, size, return the right result even if repeated elements exist? Both approaches are possible.

    Note that the order (or repetitions) of elements in the sample outputs may differ from yours. I will be testing your code using a function that tests sets for equality and disregards order and repetition, so as long as your results are equals as sets, everything will be okay.

    1. Code a function lang_in of type string -> string list -> bool where lang_in s ls returns true if string s is an element of language ls, and false otherwise.

      Show sample output

# lang_in "a" ["b";"c"];;
- : bool = false
# lang_in "a" ["a";"b";"c"];;
- : bool = true
# lang_in "a" ["c";"d";"d";"a";"a"];;

  2. Regular expressions

    In file homework3.ml, I've implemented some code to manipulate regular expressions. In particular, I've given you a function

      lang : string -> int -> int -> unit

    where lang r n max takes a regular expressionn r, an integer n, and a maximum length max, and prints out all the strings in the language corresponding to regular expression r that have length at most max. Argument n is used to limit the number of iterations for the star operator in regular expressions: for example, using n=4 means that r* in regular expressions gets interpreted as the language ⟦r⟧≤n instead of ⟦r⟧*.

    (Most of the time, you will call lang with n and max the same, basically restricting the size of the strings generated. When you have a lot of star operators in your regular expression, though, you may run into memory problems and may need to reduce n. See the sample output for regexp_d below for an example.)

    Note that function lang will not work until you complete question 1 correctly. You can still work on this question even if you haven't done question 1, of course. You just won't be able to test your answers.

    The syntax of regular expression corresponds to what we saw in class: the alphabet is restricted to be alphabetical characters (a,b,c,d,...,z,A,B,...,Z), and 0, 1, (, ), *, + all are reserved characters. For example, ab+ac is a valid regular expression, as is a(b+c) and a*(b+c)* and so on.
