Topics in Computing: Databases (Spring 2019)

Databases are omnipresent. They power everything from monolithic enterprise system to serverless web apps. Whenever there's data to be manipulated, the first impulse should be to stash it into a database first. But there are many kinds of databases, each offering different guarantees as well as different benefits.

Goals of the course

Understand the different kinds of database models that are out there, and their suitability for a given representational task:

  • relational databases
  • non-relational (NoSQL) databases:
    • key-value stores
    • document-based stores
    • column-based stores
    • graph-based stores

Understand the tradeoffs between the various database models.

Understand the various data structures and algorithms that underlie these database models

Understand how to query these models.

Understand transactional guarantees, replication, concurrency control.

Structure

The approach will be hands on. Rather than work as clients of various databases (i.e., learning how to use MySQL, or AWS Redshift, or Apache Cassandra, or MongoDB), I want us to understand databases by basically implementing them. We'll implement the essential aspects of the various database models -- in Python -- to get a feel for what each model does, from the inside out.

This may fail miserably when put to the test, but I think we'll have a lot of fun whatever happens :)
