Here is a type for generative grammars:

type symbol = string

type grammar = {
  nonterms: symbol list;
  terms: symbol list;
  rules: (string * string) list;
  initial : symbol
}

Recall that a grammar is described by a set of nonterminal symbols, a set of terminal symbols, a set of rules, and a starting (initial) symbol. The following grammar with initial nonterminal S and terminals a and b generates the language {anbn | n≥0}:

 S → ε
 S → a S b

Here is how you represent that grammar with the type above:

let anbn = {
  nonterms = ["S"];
  terms = ["a";"b"];
  rules = [("S","");
           ("S","aSb")];
  initial = "S"
}

Note that a rule P → Q is represented by a pair (P,Q) where P and Q are strings containing terminal and nonterminal symbols.

Here is a grammar generating the language {anbm | m≥n≥0}:

let anbm = {
  nonterms = ["S";"T";"B"];
  terms = ["a";"b"];
  rules = [ ("S","TB");
            ("T","");
	    ("T","aTb");
	    ("B","");
	    ("B","Bb")];
  initial = "S"
}

To check whether a string can be generated from a grammar, I provided you with a function 

    generate: int -> grammar -> string -> bool
Intuitively, generate n gr str checks whether string str can be generated by grammar gr, and returns true or false. If true, the function also prints a sequence of rewrites that shows how the string is generated. For example:

# generate 10 anbm "aabbb";;
Searching (depth 01, max width 1)
Searching (depth 02, max width 2)
Searching (depth 03, max width 3)
Searching (depth 04, max width 4)
Searching (depth 05, max width 5)
Searching (depth 06, max width 6)
   S
-> TB
-> aTbB
-> aaTbbB
-> aabbB
-> aabbBb
-> aabbb
- : bool = true

What's argument n, you ask? Well, checking if a string can be generated by a grammar is done by taking the initial symbol, and repeatedly applying all possible rules until the string is generated. That's expensive. Also, if the string cannot be generated, then this process never terminates. To control the process and prevent searching forever, rules are applied only up to depth n (that is, up to n rules can be applied in any sequence of rewrites — moreover, the maximum width of string that can be obtained during any of those rewrites is also n). This means, in particular, that if you know that a string requires K rewrites to be generated, then you need to supply a value of at least K to function generate. The tricky bit is that the largest the parameter n, the slowest the generation process, because the tree of rewrites being searched gets bigger as n increases. If your grammar gets too complicated, don't be surprised if it gets difficult to generate strings you know can be generated.
